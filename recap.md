# Genspark活用研修｜復習まとめ

## まず覚える3段階

1. 頼む: 目的、資料、完成形を渡す
2. AIが進める: 必要な機能を選び、調べる・分析する・作る
3. 人が確認して使う: 事実、機密、著作権、レイアウトを確認する

## 会話・Workflow・Agent

- 会話: 人が1回ずつ次の指示を出す
- Workflow: 最初に順番と分岐を決めて繰り返す
- Agent: ゴールに向け、AIが状況に応じて次の行動を選ぶ

最初は会話で小さく試します。手順が固まればWorkflowへ、途中判断が多い仕事だけAgentを考えます。

## Gensparkの入口

- 迷った時: Super Agent
- 表・照合・集計: AI Sheets
- 資料: AI Slides
- 共有文書: AI Docs
- 品質基準を再利用: Skill
- 順番・分岐・承認場所を再利用: Workflow

## AIエージェントの仕組み

AIエージェントはLLMだけではありません。目的と制約、LLM、Tools、Environment、Context / State / Memory、Agent loop、Harness / Permissions、人の承認を合わせた仕組みです。

### ミニ用語集

- LLM: 指示と状況を読み、次の行動を考える「頭」
- Tools: 検索、ファイル、表計算、APIなど外部へ働きかける「手」
- Environment: Webや業務システムなど処理を行う「作業場所」
- Context / State / Memory: 今回の指示、途中状態、必要に応じて後から参照する情報
- Agent loop: 計画→実行→観察→修正を繰り返す「見直し」
- Harness / Permissions: 使える道具、権限、回数、停止条件を制御する「ガード」
- 人の承認: 送信、削除、公開などの前で止める最終判断

復習ページ：仕組みはスライド6-7、入口は27-30、実演は31-36を参照してください。

## 安全に使う確認項目

- 数字、日付、固有名詞、出典を元資料と照合したか
- 個人情報、社外秘、他顧客情報が入っていないか
- 画像や文章の利用条件を確認したか
- 外部送信、削除、公開の前に人が承認したか
- 相手が見る形式で文字切れ・重なりを確認したか

## 明日からの進め方

1. 繰り返している業務を1件選ぶ
2. 架空データまたは小さな安全なデータで試す
3. 良かった品質基準をSkillへ残す
4. 手順が固まったらWorkflowへ残す
5. 失敗と要確認条件も記録する

料金、Credits、利用可能機能、画面名称、Connector、管理者設定は変更される可能性があります。実施日当日に公式情報と自社の契約・運用ルールを確認してください。
