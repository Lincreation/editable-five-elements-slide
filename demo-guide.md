# Genspark講師実演ガイド

## ねらい

前半は2つのシートを照合し、AIが曖昧な行を勝手に確定せず、人へ確認を返せるかを見せます。後半は同じ題材から短い説明資料を作り、ページを絞って修正します。完全自動化の速さではなく、判断基準・要確認・検算・部分修正の設計を扱います。

## 使用ファイル

`demo-data/Genspark研修_飲食店週次支出照合デモ.xlsx`

- `部門別支出`: 表示順、部門コード、判断基準、予算、前週実績
- `請求書明細`: 順不同の請求書13行。表記揺れ、詳細不足、再送候補を含む
- すべて研修用の架空データ

## 実演プロンプト

```text
部門別支出シートの表示順と判断基準に従い、請求書明細を分類・並べ替えてください。
部門コード、部門名、判定理由、確信度、要確認を追加してください。
判断材料が不足する行は推測で確定せず要確認とし、確認質問を作ってください。
元の請求書IDと金額は変更しないでください。
重複候補を検出し、処理前後の総額を照合してください。
```

## 進め方（30分）

### AI Sheets（20分）

1. 2分: スライド31-32で入力と狙いを説明する
2. 3分: ファイルをAI Sheetsへ添付し、2シートを確認する
3. 6分: 実演プロンプトを実行する
4. 6分: 結果の理由・確信度・要確認・重複候補・確認質問・総額を確認する
5. 3分: `.xlsx`で出力し、要確認行だけ人へ渡す流れを説明する

### AI Slides（10分）

1. 2分: 目的・対象者・結論・根拠資料を指定する
2. 4分: Professional、Guide Mode ONで3ページの構成案または初稿を作る
3. 3分: 1ページだけを選び、見出しまたは情報量を部分修正する
4. 1分: 出力後に文字切れ・数字・出典を確認する

実演プロンプト例:

```text
飲食店の店長向けに、AIで分類した請求書を人が確認する手順を3ページの研修資料にしてください。
結論は「自動分類をそのまま確定せず、要確認行・重複候補・処理前後の総額を人が確認する」です。
Professionalの落ち着いたデザインにし、1ページ1メッセージ、数字には根拠を付けてください。
添付資料にない事実や効果は推測で追加しないでください。
まずページ構成だけを提示し、承認後に生成してください。
```

AI Slidesの合格条件：

- 1ページ目に「AIの分類をそのまま確定しない」という結論がある
- 2ページ目に`INV-3112`の要確認と`INV-3104-再送`の重複候補がある
- 3ページ目に、理由・原票・総額858,640円を人が確認する手順がある
- 添付結果にない効果・比率・市場数値を追加していない
- 1ページ1メッセージで、投影時に本文が読める
- 出力先と共有権限は当日の契約プラン・管理者設定を確認する

## 合格条件

- 部門別支出の表示順に並ぶ
- 部門コード、部門名、判定理由、確信度、要確認が追加される
- `INV-3112 / 研修取引先L（詳細不明） / 緊急対応一式` を詳細不足として要確認にする
- `INV-3104-再送` を重複候補として検出する
- 元の請求書IDと金額を変更しない
- 処理前の13行合計 `858,640円` と結果を照合する
- 確定できない行について確認質問を作る

## 講師が強調すること

- AIの分類理由を読める形にする
- 確信度が低い行は人へ返す
- 「重複候補を除外した金額」と「元データの総額」を混同しない
- 合計が合っていても分類が正しいとは限らないため、理由と原票を確認する
- 実データでは個人情報・取引条件・共有範囲を事前に確認する

## うまくいかない場合

- シート名が認識されない: `部門別支出` と `請求書明細` を明示する
- 曖昧な行を確定した: 「詳細不足は推測で確定せず、要確認に戻す」と再指示する
- 総額が変わった: 元の請求書IDと金額を保持し、行単位で差分を出させる
- 出力列が多すぎる: 必須列を部門コード、部門名、判定理由、確信度、要確認に限定する
- AI Slidesの生成が長い: `demo-data/AI-Slides_週次支出報告_バックアップ.html`を表示し、「2ページの見出しを『人が確認する2件』へ変える。本文と配色は変えない」という修正指示を口頭で説明する。固定HTMLの編集操作は行わない
- 構成が広がりすぎた: 3ページに限定し、目的・対象者・結論・根拠資料を再提示する
