# 30分画面共有｜実演ガイド

## 目的

一つの仕事を「成果物にする→良いやり方をSkillにする→固まった手順をWorkflowで検証する」までつなげます。機能一覧の巡回ではなく、企業で再利用する流れを見せます。

## 0〜2分｜ホームと入口

1. ホーム画面またはNewメニューを開く。
2. Super Agent、AI Slides、AI Docs、AI Sheets、画像生成の入口を示す。
3. 完成物が決まっている時は専門機能、進め方から相談する時はSuper Agentと説明する。

## 2〜11分｜AI Sheetsで一件を成功させる

使用ファイル：`demo-data/Genspark研修_飲食店週次支出照合デモ.xlsx`

> 部門別支出シートの表示順と判断基準に従い、請求書明細を分類・並べ替えてください。部門コード、部門名、判定理由、確信度、要確認を追加してください。判断材料が不足する行は推測で確定せず要確認とし、確認質問を作ってください。元の請求書IDと金額は変更しないでください。重複候補を検出し、処理前後の総額を照合してください。

確認すること：

- 13件、入力858,640円、出力858,640円、差額0円。
- `INV-3112`は詳細不足、`INV-3104-再送`は重複候補。
- 元のIDと金額が変わらず、判断理由と確認質問が残る。
- XLSXで出力し、開き直して確認できる。

開始4分で結果が出なければ、`demo-data/AI-Sheets_分類結果_バックアップ.xlsx`の`投影用サマリー`へ切り替えます。

## 11〜18分｜主要成果物へ展開する

AI Sheetsで確認した次のサマリーをコピーします。機能間の直接連携は前提にしません。

> 13件 / 総額858,640円 / 例外2件。詳細不足はINV-3112、重複候補はINV-3104-再送。数値は元データと照合済み。

### 11〜15分｜AI Slides

> 以下の確認済みサマリーをもとに、店舗責任者向けの週次支出報告を3ページで作成してください。1ページ目は表紙、2ページ目は数値と要点、3ページ目は2件の確認事項にしてください。結論先行で、数値を変更せず、不明点は推測で補わないでください。

1. Professional / CreativeとGuide Modeの選択を示す。
2. 構成案を確認してから生成する。
3. 2ページ目だけを指定して部分修正する。
4. 2分30秒で初稿が出なければ`demo-data/AI-Slides_週次支出報告_バックアップ.html`へ切り替える。

### 15〜16分｜AI Docs

1. 同じ確認済みサマリーを新しいAI Docsの指示へ貼る。
2. Rich Text / Markdown、AI Edit、Save Point、出力場所を示す。
3. 完成は待たず、短い週次共有文の形を確認する。

### 16〜18分｜画像生成・編集

1. 「週次支出報告の表紙に使う16:9画像」と用途から指定する。
2. 比率、構図、色、避けたい文字・ロゴを伝える。
3. 部分修正の入口と、人物・文字・権利を人が確認する点を示す。

## 18〜23分｜成功した一件をSkillにする

1. 講師用環境だけでSkillsから`My Own`を開き、`+ New Skill`→`Create for myself`を選ぶ。
2. 名前を「週次支出チェック」とする。
3. 入力を「部門基準＋請求書明細」、出力を「分類結果＋例外＋総額照合」に固定する。
4. 判断基準として「推測確定しない」「IDと金額を変えない」「差額0円」を渡す。
5. Tool permissionを確認し、研修では不要なConnectorを付けない。
6. 研修用Excelで一度試し、`My Own`へ保存する。受講者は同じ内容をSkill Cardへ記入する。

説明する境界：Skillは品質を保証するものではありません。同じ入力・構造・判断基準を再利用し、ばらつきを減らす仕組みです。参加者は新規Skill作成や共有を行わず、Team公開、Peer Share、Public Linkは講師も行いません。

## 23〜30分｜WorkflowをTest Runする

1. Workflowsから`New Custom Workflow`を開く。
2. 次の研修用指示を入力する。

> 毎週月曜9時に週次支出確認を開始し、研修用の支出データを読み、分類結果と例外一覧を作る流れを設計してください。例外がある場合は人の確認待ちにし、外部送信・削除・公開は行わないでください。

3. Triggerが1つ、Actionが何を読む・作るのかを確認する。
4. `Test Run`で模擬データを使い、実送信や実更新が起きないことを説明する。
5. 正常系ではレビュー用保存、例外2件では`Pending Confirmation`、差額不一致または必須列欠落では停止する想定を見せる。
6. Run Historyは画面で示し、手動復旧はWorkflow Card上の研修設計として説明する。
7. `Turn On`は押さず、ライブ実行との違いを確認して終える。

受講者用の代替証拠：`demo-data/Genspark研修_Workflowテスト証拠.xlsx`。これはGenspark Run Historyの実ログではなく、正常系・例外系・Hard Stopを表上で試したテーブルテストです。

## 代替進行

| 障害 | 切り替え |
| --- | --- |
| ログイン不可 | スライド26と参加者ハンドアウトで一連の流れを説明 |
| AI Sheets生成待ち | `AI-Sheets_分類結果_バックアップ.xlsx` |
| AI Slides生成待ち | `AI-Slides_週次支出報告_バックアップ.html` |
| 画像生成待ち | 用途・比率・部分修正の指示だけ説明 |
| Skills作成不可 | 参加者ハンドアウトのSkill設計カードを完成させる |
| Workflows権限なし | Workflow設計カードでTrigger・Action・停止地点を作る |

## 実演終了条件

- 成果物を作るだけでなく、SkillとWorkflowへ育てる順序を説明できた。
- Skill Cardの入力契約・正本・判断・合格・NG・Owner・Versionを見せた。
- Workflow CardのTrigger・Action・停止・保存先・履歴・手動復旧・最小権限を見せた。
- 正常系・例外系のTest Recordと本番化可否を見せた。
- Team公開、Connector接続、Turn On、外部送信を行っていない。
