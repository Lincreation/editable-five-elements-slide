設計する4項目

成果物：最後に何を残すか

復習｜覚えるのは「頼む→進める→確認する」

  1. 頼む：目的、資料、完成形を渡す
  2. AIが進める：必要な機能を選び、調べる・分析する・作る
  3. 人が確認して使う：事実、機密、権利、レイアウトを確認する

入口の使い分け

  • 迷った時はSuper Agent。成果物が決まっていればAI Sheets / Slides / Docsへ直接入る
  • Skillは「何を、どの品質で作るか」を再利用する
  • Workflowは「どの順番・分岐・承認で進めるか」を再利用する

AIエージェントのミニ用語集

LLM
指示と状況を読み、次の行動を考える「頭」
Tools
検索、ファイル、表計算、APIなどへ働きかける「手」
Environment
Webや業務システムなど、処理を行う「作業場所」
Context / State / Memory
今回の指示、途中状態、後から参照する情報
Agent loop
計画→実行→観察→修正を繰り返す「見直し」
Harness / Permissions
道具、権限、回数、停止条件を制御する「ガード」
人の承認
送信、削除、公開などの前で止める最終判断
