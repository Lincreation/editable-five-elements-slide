# 30分演習｜一件をSkillとWorkflowへ育てる

## ゴール

架空Excelで一件を確認し、その成功条件をSkill Card・Workflow Card・Test Recordへ変換します。

- AI Sheets: `13件 / 入力858,640円 / 出力858,640円 / 差額0円 / 例外2件`
- Skill Card: 入力契約・正本・判断・合格・NG・Owner・Versionが書ける
- Workflow Card: Trigger・Action・停止・保存・履歴・復旧・権限が書ける
- Test Record: 正常系・例外系・Hard Stopの期待結果と本番化可否が書ける

使用ファイル：`participant/Genspark研修_飲食店週次支出照合デモ.xlsx`

## 1. 入力とゴールを確認する｜0〜4分

- [ ] 配布された研修用Excelである
- [ ] 個人情報・顧客情報・機密が含まれていない
- [ ] 元の請求書IDと金額を変更しない
- [ ] 合格値は13件・858,640円・差額0円・例外2件

## 2. AI Sheetsまたは完成例で照合する｜4〜9分

> 部門別支出シートの表示順と判断基準に従い、請求書明細を分類・並べ替えてください。部門コード、部門名、判定理由、確信度、要確認を追加してください。判断材料が不足する行は推測で確定せず要確認とし、確認質問を作ってください。元の請求書IDと金額は変更しないでください。重複候補を検出し、処理前後の総額を照合してください。

- [ ] `INV-3112`が詳細不足の要確認になっている
- [ ] `INV-3104-再送`が重複候補になっている
- [ ] 13件、入力858,640円、出力858,640円、差額0円
- [ ] 元データまたは完成済み例と比較し、件数・総額・例外を再確認した

生成が終わらない場合は、講師の完成済みExcelを使って確認へ進みます。

## 3. Skill Cardの空欄3項目を作る｜9〜16分

| 項目 | 書く内容 |
| --- | --- |
| 目的 | 部門基準に沿って週次支出を安全に分類する |
| 許可入力・入力契約 | 研修用の部門基準と請求書明細。必須列が欠けたら停止 |
| 正本・参照 | 配布Excelの部門別支出シートと請求書明細シート |
| 判断手順 | 部門基準で分類→理由→確信度→要確認→総額照合 |
| 出力形式 | 分類結果、判定理由、例外、確認質問、総額照合 |
| 合格条件 | 13件・858,640円・差額0円・例外2件 |
| NG・例外 | 推測確定、ID/金額変更、重複見落とし、必須列欠落 |
| Owner・Version | 担当者名／v0.1／見直し日 |

受講者は表の完成を合格とします。講師から承認済みSkillが指定された場合だけ研修用データで試行します。新規Skill作成、Team公開、Peer Share、Public Linkは行いません。

## 4. Workflow Cardの空欄3項目を作る｜16〜23分

| 項目 | 研修用の設計 |
| --- | --- |
| Trigger | 毎週月曜9時のSchedule。1 Workflowにつき1 Trigger |
| 読取・コピー | 正本を直接更新せず、研修用コピーを読む |
| Action / Skill | 分類→理由→例外一覧→総額照合。承認済みSkillだけ使用 |
| 分岐・停止 | 例外2件は確認待ち。差額不一致・必須列欠落は停止 |
| 保存先 | 承認前は研修用フォルダへレビュー用保存 |
| 履歴・通知 | Run Historyへ記録し、確認者へ通知する設計 |
| 手動復旧 | 元データを再確認し、コピーから再実行 |
| Owner・最小権限 | 担当者名／読取＋レビュー用保存のみ |

受講者は設計表の完成を合格とします。講師が停止中の研修用Workflowを用意した場合だけTest Runを確認します。Turn On、Connector認証、外部送信は行いません。

## 5. Test Recordを照合する｜23〜27分

`participant/Genspark研修_Workflowテスト証拠.xlsx`を開きます。これはGenspark Run Historyの実ログではなく、停止設計を正常系・例外系・Hard Stopで試したテーブルテストです。

| ケース | 期待結果 | 実結果・証拠 | 判定 |
| --- | --- | --- | --- |
| 正常系：3件・600,000円・差額0円・例外0件 | 研修用のレビュー用保存まで進む | 差額0円・例外0件・必須列OK | Pass / Fail |
| 例外系：3件・600,000円・例外2件 | Pending Confirmationで止まる | 差額0円・例外2件・必須列OK | Pass / Fail |
| Hard Stop：出力599,000円・必須ID欠落 | 保存せず停止する | 差額1,000円・必須ID欠落 | Pass / Fail |

本番化可否：`不可（研修中）`　改善点：＿＿＿＿＿＿＿＿＿＿＿＿＿＿＿＿

## 6. ペアレビュー・質問｜27〜30分

1. 相手のカードだけで、許可された入力と合格条件を説明できますか。
2. 例外時にどこで止まり、誰が承認するか説明できますか。
3. Run Historyと手動復旧の方法がありますか。

## 修了条件

- 一回限りの依頼、Skill、Workflowを使い分けられる。
- Skill Cardに入力契約・合格条件・例外・Owner・Versionがある。
- Workflow Cardに停止・承認・履歴・手動復旧・最小権限がある。
- 正常系・例外系・Hard StopのTest Recordがあり、本番化不可と判断できる。
- Team公開、外部接続、Turn Onを行っていない。
