# Genspark基本研修｜講師ガイド

## 研修の芯

受講者に覚えてもらうのはボタン配置ではなく、次の判断軸です。

- GeminiとGensparkは、正本・保存先・権限・作業場所で選ぶ。
- 完成物が決まっていれば、AI Slides・Docs・Sheets・画像生成へ直接入る。
- 迷う、または複数工程ならSuper Agentへ相談する。
- まず一件を成功させ、Skillで作り方をそろえ、Workflowで実行手順を回す。
- Skillは入力・出力・判断基準、WorkflowはTrigger・Action・Test Run・停止地点で設計する。
- AIの完成品らしさと正しさは別。最後は人が確認する。

## 前日・開始30分前

- `index.html`、参加者版、Excel、AI Sheets完成版、AI Slides完成例を別タブで開く。
- 実演Excelは添付できる状態まで確認し、生成は開始しない。
- 出席・理解度確認のURLまたはQRを当日資料へ差し替える。
- 画面共有は通知を切り、他案件のタブとファイルを閉じる。

## 60分説明を60分にする確認ワーク

- 0:10：隣の人に「GeminiとGensparkを何で選ぶか」を30秒で説明する。
- 0:20：AIエージェントを「頭・手・作業メモ・止める仕組み」で1分説明する。
- 0:28：自分の業務を、目的・対象者・参考資料・完成形式・制約の5要素で45秒メモする。
- 0:43：Slides / Docs / Sheets / 画像のどれを使うか、理由と一緒にチャット投稿する。
- 0:55：同じ指示を繰り返す仕事と、同じ手順を繰り返す仕事を分類し、停止条件を一つ書く。

問いかけ、挙手・チャット確認、説明の言い換えを含めて60分です。講師の一方通行の読み上げだけで60分を埋めません。

## デモ打ち切り基準

- AI Sheetsの結果が開始4分で出なければ完成済みExcelへ切り替え、結果確認に最低3分を残す。
- AI Slidesの初稿が開始2分30秒で出なければ完成済みHTMLへ切り替える。
- SheetsからSlides / Docsへは、確認済みサマリーをコピーして新規指示へ貼る。直接連携は前提にしない。
- 画像生成は1回だけ実行し、長引けば既存結果で「部分修正の考え方」を説明する。
- Skillの作成・試行は講師用環境のMy Ownだけ。受講者は設計カードを基本とし、Team公開、Peer Share、Public Linkは行わない。
- Workflowは模擬データのTest RunとRun History確認まで。Turn Onは行わない。
- 受講者のTest Recordは、`participant/Genspark研修_Workflowテスト証拠.xlsx`の3ケースで照合する。これはGenspark Run Historyの実ログではなく、停止設計を表上で試すテーブルテストと説明する。

## 終了条件

- 受講者が4つの主要機能を一言で説明できる。
- GeminiとGensparkを能力の有無ではなく作業環境で説明できる。
- 一回限りの依頼、Skill、Workflowを使い分けられる。
- 入力契約・正本・判断・合格・NG・Owner・Versionを含むSkill Cardを一つ作れる。
- Trigger・Action・停止・保存先・履歴・手動復旧・最小権限を含むWorkflow Cardを一つ作れる。
- 正常系・例外系・Hard Stopの期待・実結果・本番化可否を含むTest Recordを作れる。
- 配布Excelの結果について、元データ、理由、例外、総額の確認場所を説明し、XLSX出力を開き直せる。
- 出席・理解度確認を完了し、次に試す一件を一つ決める。
