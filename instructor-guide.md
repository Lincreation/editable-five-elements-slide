# Genspark基本研修｜講師ガイド

## 研修の芯

受講者に覚えてもらうのはボタン配置ではなく、次の判断軸です。

- GeminiとGensparkは、正本・保存先・権限・作業場所で選ぶ。
- 完成物が決まっていれば、AI Slides・Docs・Sheets・画像生成へ直接入る。
- 迷う、または複数工程ならSuper Agentへ相談する。
- Skillは品質、Workflowは順番と自動化を再利用する。
- AIの完成品らしさと正しさは別。最後は人が確認する。

## 前日・開始30分前

- APHDアカウントでSuper Agent、AI Slides、AI Docs、AI Sheets、画像生成、Skills、Workflowsを確認する。
- 画像機能の当日表示名を確認し、スライドの「画像生成・編集」と対応づける。
- `index.html`、参加者版、Excel、AI Sheets完成版、AI Slides完成例を別タブで開く。
- 実演Excelは添付できる状態まで確認し、生成は開始しない。
- 出席・理解度確認のURLまたはQRを当日資料へ差し替える。
- 画面共有は通知を切り、他案件のタブとファイルを閉じる。

## 50名運営

- 質問はMeetチャットまたは指定フォームへ一本化する。
- 操作状況は `○ 完了 / △ 途中 / ? 質問` で投稿してもらう。
- 1:35、1:42、1:50に全体停止し、現在地を合わせる。
- 個別画面共有は原則1名まで。共通する質問を優先して全体回答する。
- 生成待ちの人は、演習シートで確認基準と修正指示を先に書く。
- 両日とアーカイブで、出席確認と理解度確認の完了条件を同じにする。

## 60分説明を60分にする確認ワーク

- 0:10：隣の人に「GeminiとGensparkを何で選ぶか」を30秒で説明する。
- 0:22：AIエージェントを「頭・手・作業メモ・止める仕組み」で1分説明する。
- 0:31：自分の業務を、目的・対象者・参考資料・完成形式・制約の5要素で1分メモする。
- 0:47：Slides / Docs / Sheets / 画像のどれを使うか、理由と一緒にチャット投稿する。
- 0:55：SkillとWorkflowの例を一つずつ、近くの人と2分で言い換える。

問いかけ、挙手・チャット確認、説明の言い換えを含めて60分です。講師の一方通行の読み上げだけで60分を埋めません。

## デモ打ち切り基準

- AI Sheetsの結果が開始5分で出なければ完成済みExcelへ切り替え、結果確認に最低5分を残す。
- AI Slidesの初稿が開始2分30秒で出なければ完成済みHTMLへ切り替える。
- SheetsからSlides / Docsへは、確認済みサマリーをコピーして新規指示へ貼る。直接連携は前提にしない。
- 画像生成は1回だけ実行し、長引けば既存結果で「部分修正の考え方」を説明する。
- Workflowsは有効化しない。画面とTest Run、Pending Confirmationの説明で終える。
