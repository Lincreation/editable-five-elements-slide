# Genspark活用研修｜講師進行ガイド

## 研修全体

- 合計120分
- 講義60分
- 講師実演30分（AI Sheets 20分＋AI Slides 10分）
- 参加者操作・質問30分
- 到達点: 受講者が「AIに任せる範囲」「人が確認する場所」「再利用する品質基準と手順」を一つの業務に置き換えられる

## 事前準備

- `index.html?present=1`を投影し、通常の`index.html`でページ送り・講師台本表示を確認する
- Gensparkへログインし、AI SheetsとAI Slidesを別タブで開く
- 講師用の`demo-data/Genspark研修_飲食店週次支出照合デモ.xlsx`と、受講者配布用の`participant/`フォルダを用意する
- 受講者のアカウント、利用可能機能、Credits、ファイルアップロード可否を確認する
- ライブ生成が失敗した場合に備え、`demo-data/AI-Sheets_分類結果_バックアップ.xlsx`と`demo-data/AI-Slides_週次支出報告_バックアップ.html`を開けるようにする
- 実データは使わず、研修用の架空データだけを使う

### 開始10分前のリハーサル

1. `index.html`の1→44、44→1、目次から27・31・44への移動を確認する
2. AI Sheetsへ研修Excelを添付できるところまで確認し、実行前で止める
3. AI Slidesを開き、Professional / Guide Mode / Qualityの当日表示を確認する
4. 2つのバックアップ成果物を開き、Excelの13明細・858,640円と、HTMLの3ページを確認する
5. AI Sheets実演開始から10分30秒で結果が出なければ完成済みExcelへ、AI Slides実演開始から6分30秒で初稿が出なければ完成済みHTMLへ切り替える
6. モードやQualityの選択肢が見つからない場合は、探す時間を60秒で打ち切り、当日画面のまま目的・対象者・結論の入力へ進む

### 37-43の扱い

37-43はAI Slidesの補足・復習用です。10分実演中に全ページを説明すると操作時間がなくなるため、通常進行では表示しません。スライド36を説明したら、目次から`44`を選び参加者操作へ進みます。質問が出た時、または受講後の復習で37-43を使用します。

### 後方席へ見せる時

- スライド12はExcel画面内の小さい文字を読ませず、右側の「話しかける」「使いたい列を指定」の2点を説明する。操作時はライブ画面を125〜150%へ拡大する
- 完成済みAI Sheetsは`投影用サマリー`を150%で表示し、13明細・入力858,640円・出力858,640円・差額0円・例外2件を先に示す。判定理由と確認質問の細部は例外行だけを拡大して読む

## AI Sheets実演｜20分

1. 0-2分: スライド31-32で入力と狙いを説明
2. 2-5分: Excelを添付し、2シートを確認
3. 5-11分: スライド33のプロンプトを実行
4. 11-17分: 部門順、判定理由、確信度、要確認、重複候補、総額を確認
5. 17-20分: `.xlsx`出力と、人へ渡す要確認行を説明

講師が見る合格条件：

- `INV-3112 / 研修取引先L（詳細不明） / 緊急対応一式`を推測で確定せず、確認質問を作る
- `INV-3104-再送`を重複候補として残す
- 元の請求書IDと金額を変えない
- 処理前後の総額が858,640円で一致する

## AI Slides実演｜10分

1. 0-2分: スライド35-36で目的・対象者・結論・参考資料を説明
2. 2-3分: AI Slidesへ移動し、Professional / Guide Mode / Qualityの現在画面を確認する
3. 3-5分: 下記プロンプトとAI Sheetsの結果を渡す
4. 5-8分: 生成中に、ページごとの役割と出典の残し方を説明
5. 8-9分: 見出し1件だけを指定して部分修正
6. 9-10分: 事実、機密、著作権、レイアウトと、出力先の権限を確認

実演プロンプト：

```text
飲食店の店長向けに、AIで分類した請求書を人が確認する手順を3ページの研修資料にしてください。
結論は「自動分類をそのまま確定せず、要確認行・重複候補・処理前後の総額を人が確認する」です。
Professionalの落ち着いたデザインにし、1ページ1メッセージ、数字には根拠を付けてください。
添付資料にない事実や効果は推測で追加しないでください。
まずページ構成だけを提示し、承認後に生成してください。
```

合格条件は`demo-guide.md`の「AI Slidesの合格条件」を使用します。生成が間に合わない場合は、バックアップHTMLへ切り替え、どの修正指示を出すかを口頭で実演します。

## 参加者操作・質問｜30分

受講者は`participant/handout.html`を使い、講師は進行用の`exercise-sheet.md`を参照します。講師は答えを先に示さず、次の時刻で全体を止めます。

- 0-5分: ゴールと安全ルールを確認し、Excelを添付する
- 5-17分: プロンプトを実行し、必要なら一度だけ追加指示する
- 17分: 全員が生成を止める。未完の人も完成済みExcelとの比較へ進む
- 17-22分: 合格条件で自己確認する
- 22-26分: 1名または1組が判断基準と要確認行を共有する
- 26-30分: 質問と明日の1件を決める

## 代替進行

| トラブル | 代替 |
| --- | --- |
| アカウント未準備 | 2人1組で画面を共有し、操作担当と確認担当に分ける |
| AI Sheetsが使えない | Excelを開き、`exercise-sheet.md`の「紙上設計」で分類基準・要確認条件を書く |
| 生成が長い | 講師PCの完成済み結果へ切り替え、プロンプトと確認手順を解説する |
| 画面表記が違う | 機能名を断定せず、目的・入力・出力・確認場所で案内する |
| Credits不足 | 実行せず、プロンプトと完成条件の設計演習へ切り替える |
| ファイルをアップロードできない | 画面共有で講師が1回だけ実行し、受講者は判断基準を書き換える |
| 参加者全員がGensparkを利用できない | 「全員、紙上設計へ切り替えます」と宣言し、演習シートの入力・処理・人の確認・成果物を記入する |

## 終了条件

- 受講者が、自分の業務1件について入力、AIの役割、人の確認、成果物を言える
- 研修Excelで、曖昧行を推測で確定しない理由を説明できる
- SkillとWorkflowを品質基準と手順に分けて説明できる
- 明日試す小さな1件を決めている
