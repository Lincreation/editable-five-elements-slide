# Genspark基本研修｜6周自己改善ログ

- 実施日: 2026-07-20
- 評価基準: `qa/evaluation-criteria.md`
- 評価体制: 目的・事実、学習設計、デザインの独立3監査＋別プロセスCodex＋主担当統合
- 対象: 28枚、60分台本、30分実演、30分演習・質問、講師資料、参加者資料、2種の配布Excel

## Round 1｜目的を再定義

- 抽象的なAgent論を主役から外し、企業研修の目的をGeminiとの使い分け、主要4機能、Skills / Workflows、安全な再現へ固定した。
- 120分をTalk 60、Show 30、Do / Q&A 30へ明確に分けた。
- 「この研修後」という曖昧な見出しをやめ、研修の目的と4つの到達目標を冒頭で示した。

## Round 2｜公式情報と技術説明を補正

- GeminiもSlides / Sheetsを扱えるため、優劣ではなく正本・保存先・権限・作業場所で比較する形へ変更した。
- Genspark Help Center、Google公式ヘルプ、Anthropicの一次情報で現行名称と役割を確認した。
- AgentをLLM、Tools、Context、Loop、Guardrailsへ分解し、業務上の安全境界まで説明できるようにした。

## Round 3｜60分で教えられる順番へ再設計

- 28枚を、目的→違い→Agent→基本操作→主要4機能→Skills / Workflows→安全→まとめの順へ全面再構成した。
- 各スライドへ進行目安と話す内容を埋め込み、外部台本と1対1で同期した。
- カード、工程、比較、作例のレイアウトを使い分け、章扉の反復を減らした。

## Round 4｜実演・演習・配布物を同期

- 主演習Excelを13件・858,640円・例外2件へ固定し、AI Sheets→Slides / Docs / 画像→Skill→Workflowの一連の実演にした。
- `INV-3112`と`INV-3104-再送`を全資料で同じ例外として扱った。
- AI SheetsとAI Slidesの完成済みバックアップ、デモ打切時刻、紙上代替、参加者ハンドアウトを追加した。

## Round 5｜Skills / Workflowsを企業研修の中心へ変更

- 「機能を知る」で終わらず、「成果物を作る → Skillで作り方をそろえる → Workflowで安全に回す」を研修の芯にした。
- Skill CardにINPUT / LOGIC / QUALITY / OUTPUT、Owner、Versionを置いた。
- Workflow CardにTrigger、Action、停止、承認、保存、履歴、手動復旧、最小権限を置き、Test RunとTurn Onを分離した。
- 受講者、Owner、管理者の操作範囲を分け、本番正本・公開・外部送信・Connector・Turn Onを研修から除外した。

## Round 6｜証拠と50名運営を実証

- 監査で見つかった「正常系の入力がない」「演習が過密」「390pxで横にはみ出す」を修正した。
- 正常系、例外系、Hard Stopの3ケースを持つ`Genspark研修_Workflowテスト証拠.xlsx`を作り、期待結果、模擬実行結果、証拠、Pass、本番化不可を残した。
- これはGenspark Run Historyの実ログではなく、研修用テーブルテストであることを明示した。
- 演習を4＋5＋7＋7＋4＋3分へ再配分し、参加者が埋める項目をSkill Card 3項目、Workflow Card 3項目へ圧縮した。
- スライド、台本、講師ガイド、実演ガイド、演習シート、ハンドアウト、README、配布マニフェストを再同期した。
- 390pxの長いボタン、グリッド、長文、表の幅制約を修正し、ページ全体の横方向欠けを解消した。

## 最終判定

- `qa/verify.mjs`は108項目合格。28枚、台本28節、Talk 60、Show 30、Do / Q&A 30が一致。
- 目的・事実100/100、学習設計100/100、デザイン96/100。すべてP0 0件、P1 0件。
- 別プロセスCodexも最新版の時間配分を再照合し、98/100以上、P0 0件、P1 0件、必須ゲート7/7と判定した。
- 必須7ゲートは7/7合格。
- Skills / Workflowsは講義、実演、演習、持ち帰り成果物のすべてで中心に置かれた。
- Vercel本番公開は依頼対象外のため行わない。
