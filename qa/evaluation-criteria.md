# 評価基準｜100点

合格条件：90点以上、P0 / P1が0件。

| 観点 | 配点 | 合格条件 |
| --- | ---: | --- |
| 研修目的との一致 | 15 | Gemini差分、主要4機能、Skills、Workflowsが「安全な再現」に収束する |
| 事実・主要機能 | 10 | 公式名称と研修用設計を区別し、Slides、Docs、Sheets、画像、Agentを正確に説明する |
| 60/30/30の学習設計 | 10 | Talk 60、Show 30、Do / Q&A 30で実施可能 |
| Skill Card | 15 | 目的、入力契約、正本、判断、出力、合格、NG、Owner、Versionを参加者が作れる |
| Workflow Card・安全 | 20 | Trigger、Action、停止、承認、保存、履歴、復旧、最小権限を設計し、Turn Onしない |
| Test Record・演習 | 15 | 正常系・例外系・Hard Stopの期待・実結果・本番化可否を残せる |
| 50名運営・代替進行 | 10 | 停止時刻、ペアレビュー、権限なし・生成待ちの同等成果物がある |
| 表示・資料同期 | 5 | 28枚、台本、ガイド、配布物の時間・用語・表示が一致する |

## 必須ゲート

次の7点が1つでも欠ければ、点数に関係なく不合格です。

- 外部送信・公開・Turn Onの前に人の承認がある
- 本番の正本を直接更新しない
- 例外時に確認待ちまたは停止する
- Skill / WorkflowのOwnerがいる
- Run Historyまたは同等の記録がある
- 手動復旧方法がある
- 正常系と例外系の2種類以上をテストする

## P0

- HTMLが開かない、操作できない
- 機密・個人情報・他顧客情報が混入
- 実データ投入、自動送信、本番更新を参加者へ促す
- WorkflowのTurn Onや外部送信を研修中に促す
- 承認・停止なしで本番化する設計
- Skill Card・Workflow Card・Test Recordのいずれかがない
- Gemini/Gensparkの明白に誤った比較

## P1

- 必須4機能の欠落
- Skill CardまたはWorkflow Cardの必須項目が欠ける
- 正常系・例外系テストのどちらかがない
- 120分を超える
- 演習ファイルが開かない
- 主要文字の切れ、重なり、投影不能な小ささ
- スライドと講師ガイド、演習、配布物の不一致
- 旧時間配分・旧用語と新設計が混在する

## 目視項目

- 全28枚を1枚ずつ1920×1080で確認する
- 文字干渉、画面端、意図した配置、色コントラストを確認する
- 章ごとの視覚差、冒頭と結論の対応、実演への切り替えを確認する
- 3成果物がPC・390px幅・A4印刷・紙上代替で記入できることを確認する
