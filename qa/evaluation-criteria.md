# Genspark活用研修｜100点評価基準

## 合格ライン

- 総合92点以上
- P0（実施不能・重大な事実誤認・情報漏えい）0件
- P1（学習目標未達・時間破綻・顧客向け不適切表記）0件
- 全スライドの重大な文字切れ・意図しない重なり0件
- ブラウザのJavaScriptエラー0件、主要操作の失敗0件
- Excelの数式エラー0件、デモ用の曖昧行・重複候補・合計値が再現できる

## 優先度

| 優先度 | 定義 | 対応 |
| --- | --- | --- |
| P0 | 研修が成立しない、危険な誤情報、機密・内部情報の露出 | 直ちに修正。残存時は納品不可 |
| P1 | 受講者が誤解する、時間内に終わらない、実演が止まる、顧客向けとして不自然 | 当該ラウンド内で修正 |
| P2 | 理解・可読性・講師運用を明確に改善する | 原則修正。見送る場合は理由を記録 |
| P3 | 好みや微細な磨き込み | 効果と回帰リスクを比較して判断 |

## 採点基準

### 1. 事実の正確性と出典追跡性｜20点

- 公式情報で確認できる機能名・仕組み・制約だけを断定している（6点）
- 料金・Credits・プラン・画面仕様など変わりやすい事項に確認日または「当日確認」がある（4点）
- AIエージェントを、目的・制約、LLM、Tools、Environment、Context / State / Memory、Agent loop、Harness / Permissions、人の承認で正確に説明できる（6点）
- スライド・台本・調査ソースから一次情報へたどれる（4点）

### 2. 学習目標・物語・120分設計｜20点

- 研修の目的と、研修後に受講者ができることが冒頭で明確（4点）
- 講義60分、講師実演30分、参加者操作・質問30分が破綻なくつながる（5点）
- 各章が前章の問いを受け、最後の業務適用へ累積的につながる（4点）
- Talk / Show / Doが区別され、説明だけで終わらない（4点）
- 終了時に受講者の次アクションと成功条件が明確（3点）

### 3. 実演・ハンズオンの実行可能性｜20点

- AI Sheets 20分、AI Slides 10分の入力・操作・見せる結果が具体的（5点）
- 参加者操作30分の入力例、制限時間、完成条件、共有観点が具体的（5点）
- デモExcelに曖昧行、重複候補、合計照合、人の確認ポイントが再現される（4点）
- 権限不足、画面差分、通信不調、処理待ちに対する代替進行がある（3点）
- 講師ガイドだけで別の講師も実施できる（3点）

### 4. 視覚・可読性・アクセシビリティ｜15点

- すべての見出しが、そのページの主張を自然な日本語で伝える（4点）
- 1920×1080で文字切れ、意図しない重なり、画面外配置がない（4点）
- 文字サイズ、行長、余白、コントラストが投影環境で読める（3点）
- 章扉、概念、実演、演習、まとめの見分けがつく（2点）
- 色だけに依存せず、番号・ラベル・形でも意味が分かる（2点）

### 5. 成果物間の整合・編集性・再現性｜15点

- スライド番号・目次・内部データ・講師台本の件数とタイトルが一致（4点）
- スライド、講師ガイド、演習シート、recap、デモガイド、調査ソースに矛盾がない（4点）
- 編集モード、ページ移動、URL直リンク、講師台本表示が動作する（3点）
- 再生成・再検証手順がREADMEまたはQAに残る（2点）
- Excelの入力・計算・検証箇所が編集可能で追跡できる（2点）

### 6. 安全性・顧客引き渡し品質｜10点

- 架空データを使用し、他顧客名、ローカルパス、内部作業メモを見せない（3点）
- Prompt Injection、最小権限、外部送信・削除前の承認を説明する（2点）
- 最終的な事実、機密、著作権、レイアウトを人が確認する設計（2点）
- 顧客が受け取った時に、資料名・使い方・次アクションが自然に分かる（2点）
- 公開範囲が明確で、明示指示のない外部公開を行っていない（1点）

## 各ラウンドの固定手順

1. 3名が独立に全範囲を採点し、P0〜P3の指摘を出す
2. 主担当が指摘を採用・却下し、理由を記録する
3. 採用指摘を成果物へ反映する
4. 全スライドを再レンダリングし、全ページを目視する
5. ブラウザ操作、件数整合、Excel数式・表示を検証する
6. 100点で再採点し、変更点・未採用指摘・残リスクを記録する

各ラウンドで指摘が出なかった場合も、評価結果と「変更なし」の根拠を記録する。点数を上げるための無意味な変更は行わない。
